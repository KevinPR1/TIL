# Continuous Integration

L'intégration continue est un ensemble de pratiques permettant vérifier à chaque modification de code source 
que le résultat des modifications ne produit pas de régression dans l'application développée



Le principal but de cette pratique est de détecter les problèmes d'intégration au plus tôt lors du développement. De plus, elle permet d'automatiser 
l'exécution des suites de tests et de voir l'évolution du développement du logiciel. 
L'intégration continue est de plus en plus utilisée en entreprise afin d'améliorer la qualité du code et du produit final
